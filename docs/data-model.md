# KidPlan Data Model

The backend store is a single Google Sheet with 6 tabs. The GAS API is the only writer; the frontend never talks to Sheets directly. Reads open the Sheet through a single `openSheet_()` chokepoint; writes go through the `upsertRow_`/`deleteRow_` helpers guarded by `LockService.getScriptLock()` (released in a `finally`).

Conventions used throughout:

- Dates are ISO 8601 calendar dates (`YYYY-MM-DD`) unless noted.
- Datetimes are ISO 8601 with timezone (`YYYY-MM-DDTHH:mm:ssZ` or local offset).
- Times-of-day are 24-hour `HH:mm`.
- Booleans are stored as the strings `TRUE` / `FALSE` (Sheets native).
- IDs are short opaque strings generated by GAS (`Utilities.getUuid()` truncated, or a monotonic counter — TBD in Wave 2).
- "Nullable: yes" means the cell may be empty.

## Tab: `Days`

One row per planned day. The hub of the app — every Today and Week view query starts here.

| Column | Type | Nullable | Notes |
|---|---|---|---|
| `date` | date | no | ISO 8601 (`YYYY-MM-DD`). Primary key for the tab; one row per date. |
| `primary_tag` | enum (string) | no | Drives the day's at-a-glance color stripe. Must exist in `Tags.tag`. |
| `tags` | string | yes | Comma-separated list of secondary tags. Each value must exist in `Tags.tag`. |
| `plan_a_summary` | string | yes | Free-text human summary of the day's main plan. |
| `plan_b_pointer` | string | yes | Either a `Library.id` (referencing a fallback activity) or the literal string `custom`. If `custom`, see `plan_b_custom_text`. |
| `plan_b_custom_text` | string | yes | Free-text custom Plan B. Only populated when `plan_b_pointer = "custom"`. |
| `notes` | string | yes | Free-text scratch notes (gear to pack, who's driving, etc.). |
| `confidence` | enum (string) | no | One of `planned`, `tentative`, `locked`. Drives subtle UI cue (dotted vs solid stripe). |
| `weather_sensitive` | bool | no | `TRUE`/`FALSE`. Flag for future weather-triggered Plan B auto-suggest (roadmap). |
| `updated_at` | datetime | no | ISO 8601 with timezone. Set by GAS on every write. |

## Tab: `PlanItems`

One row per time-blocked item. These are the rows mirrored to Google Calendar events on the family calendar.

| Column | Type | Nullable | Notes |
|---|---|---|---|
| `id` | string | no | Opaque GAS-generated ID. Primary key. |
| `date` | date | no | ISO 8601 (`YYYY-MM-DD`). Foreign key to `Days.date`. |
| `kid` | enum (string) | no | One of `shared`, `elder`, `younger`. Drives per-kid lane rendering in day view. |
| `title` | string | no | Event title shown in app and on GCal. |
| `start_time` | time | no | 24-hour `HH:mm` in the family timezone. |
| `end_time` | time | no | 24-hour `HH:mm` in the family timezone. Must be > `start_time`. |
| `location` | string | yes | Free-text (also passed to GCal `location` field). |
| `gcal_event_id` | string | yes | The GCal event ID mirrored from this row. Empty until the calendar sync writes it; thereafter used for idempotent updates. |
| `tag` | enum (string) | yes | Optional. Must exist in `Tags.tag` if set. Falls back to the parent day's `primary_tag` for display. |
| `source` | enum (string) | no | One of `manual`, `photo_ocr`. Distinguishes parent-entered items from OCR-extracted ones. |
| `updated_at` | datetime | no | ISO 8601 with timezone. Set by GAS on every write. |

## Tab: `Library`

Reusable fallback activities. Powers Plan B suggestions and the Library view.

| Column | Type | Nullable | Notes |
|---|---|---|---|
| `id` | string | no | Opaque GAS-generated ID. Primary key. Referenced by `Days.plan_b_pointer`. |
| `name` | string | no | Short label (e.g. "Children's museum"). |
| `tag` | enum (string) | no | Must exist in `Tags.tag`. Used to color the library card. |
| `description` | string | yes | Free-text detail (what to bring, why it's good for a rainy day, etc.). |
| `indoor` | bool | no | `TRUE`/`FALSE`. Filters fast for "we need an indoor backup today". |
| `typical_duration_min` | number | yes | Integer minutes. Hint for scheduling, not enforced. |
| `kid_age_fit` | enum (string) | no | One of `any`, `young`, `older`, `both`. Aligns with the two-kid lane model. |
| `notes` | string | yes | Anything else (cost, hours, parking quirks). |

---

## Tab: `Tags`

Preset + user-added day-type tags. Single source of truth for valid tag values across `Days`, `PlanItems`, and `Library`.

| Column | Type | Nullable | Notes |
|---|---|---|---|
| `tag` | string | no | Primary key. Short lowercase label (e.g. `indoor`, `day trip`, `mountains`). |
| `color` | string | no | Hex color (`#RRGGBB`) used for day stripe and badges. |
| `display_order` | number | no | Integer. Lower numbers shown first in pickers. |
| `is_preset` | bool | no | `TRUE` for the shipped seed set (indoor, outdoor, day trip, mountains, playdate, chill, errands); `FALSE` for parent-added tags. |

---

## Tab: `Photos`

Archive of paper-calendar photos and their OCR reconciliation state.

| Column | Type | Nullable | Notes |
|---|---|---|---|
| `id` | string | no | Opaque GAS-generated ID. Primary key. |
| `drive_file_id` | string | no | Drive file ID for the uploaded JPG. Lives under `/KidPlan/calendar-photos/YYYY-MM-DD-HHMMSS.jpg`. |
| `uploaded_at` | datetime | no | ISO 8601 with timezone. |
| `covers_date_range_start` | date | yes | ISO 8601 (`YYYY-MM-DD`). User-tagged or parser-inferred start of the date range visible in the photo. |
| `covers_date_range_end` | date | yes | ISO 8601 (`YYYY-MM-DD`). Inclusive end of the range visible in the photo. |
| `ocr_text` | string | yes | Raw Cloud Vision `DOCUMENT_TEXT_DETECTION` output. Can be long. |
| `parsed_json` | string | yes | JSON string of the parser's structured candidate events, pre-reconciliation. |
| `reconciled` | bool | no | `TRUE` once a human has accepted/edited/rejected every candidate row; `FALSE` while still pending review. |

---

## Tab: `Settings`

Singleton key-value store. Mirrors the Script Properties set up in the setup checklist so the frontend can read them via the API without needing Script Property access.

| Column | Type | Nullable | Notes |
|---|---|---|---|
| `key` | string | no | Primary key. App-editable keys: `family_calendar_id`, `read_only_calendar_ids`, `photo_drive_folder_id`. Secrets (`API_TOKEN`, `VISION_SERVICE_ACCOUNT_JSON`) live ONLY in Script Properties, never in this tab, and are never returned by `get_settings`. |
| `value` | string | no | Stringified value. For `read_only_calendar_ids` this is a comma-separated string. |
